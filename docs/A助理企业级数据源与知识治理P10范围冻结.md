# A助理企业级数据源与知识治理 P10 范围冻结

## 阶段定位

P10 是内部试用运营与 Pilot Readiness 闸门阶段，只修改 `AICopilot`。本阶段不修改 `IIoT.CloudPlatform`、`IIoT.EdgeClient`，不启用 Real CloudReadonly，不接真实生产数据，不开放生产 Agent 查询。

P10 的目标是把 P0-P9 已形成的 SimulationBusiness、CloudReadonlySandbox、Agent、审批、产物和审计结果汇总成内部试用运营台账，并给 P11 是否可以开始规划 Real CloudReadonly 受控生产 Pilot 提供证据。

## 允许范围

- 新增内部 Trial Campaign，用于组织内部试用。
- 只允许挂载来源为 `SimulationBusiness` 或 `CloudReadonlySandbox` 的 Agent task、artifact、approval、audit 摘要引用。
- 新增风险台账，记录风险等级、类别、状态、责任人、来源引用和处理说明 hash。
- 新增 Pilot Readiness Gate，输出 `NotEvaluated / CollectingEvidence / Blocked / ReadyForP11Planning`。
- 新增 Evidence Package，输出场景数、通过/失败/阻塞数、final 产物数、审批状态、query/result hash 样例、未关闭风险和阻塞项。
- 前端只展示后端返回的 Campaign、场景运行记录、数据边界、final 状态、审批状态、风险和准入结果。

## 禁止范围

- 不修改 Cloud/Edge。
- 不启用 Real CloudReadonly。
- 不接真实 Cloud 生产数据。
- 不开放 `query_cloud_data_readonly` 给 Planner 或 Agent。
- 不新增真实 Cloud Agent 查询接口。
- 不保存完整 SQL、完整 sandbox payload、产物正文副本、token、API Key、连接串或密码。
- 不把 `ReadyForP11Planning` 展示为生产数据已启用。
- 不覆盖 P9 final 产物，不新增危险物理删除默认路径。

## Readiness Gate 阻断项

任一条件命中时，P11 准入状态必须为 `Blocked`：

- 生产读取开关被启用。
- 生产 CloudReadonly 工具可见或可执行。
- 存在 Cloud/Edge 修改。
- 存在明文密钥、连接串、token 或完整敏感 payload 输出。
- final 产物可以被覆盖。
- P9 final 锁定证据缺失。
- 存在未关闭的 High/Critical 风险。
- 附加了非 `SimulationBusiness` 或非 `CloudReadonlySandbox` 的试用证据。

## 完成条件

- Trial Campaign、Scenario Run、Risk Issue 可持久化。
- Agent task 只能以引用方式挂载，且来源边界只允许 `SimulationBusiness` 和 `CloudReadonlySandbox`。
- Pilot Readiness Gate 能输出阻塞项、警告和指标。
- Evidence Package 只输出摘要和 hash。
- 前端工作台能查看 Campaign、运行记录、风险、准入结果和证据包摘要。
- P10 验收脚本生成 `docs/enterprise-trial-operations-p10-latest.md`。
