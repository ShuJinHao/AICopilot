# A助理企业级数据源与知识治理 P14.2 范围冻结

P14.2 是 Production Pilot Operations 持久化与 P15 准入硬化阶段。它只修 P14 运营闸门的企业化缺口，不进入 P15，不扩大生产读取范围。

## 本阶段允许

- 只修改 `AICopilot`。
- 将 P14 emergency stop、incident、run ledger、GA readiness assessment 从内存态升级为 `AiGatewayDbContext` 持久化。
- 新增独立 `ProductionOperations` 聚合和 EF migration。
- 强化 P15 readiness：必须同时具备 P12 completed run 和 P13 completed run 证据。
- 继续使用 hash-only 运营台账，不持久化 rows/raw payload。
- 补充 P14.2 backend tests、frontend smoke、scope guard 和 acceptance 报告。

## 本阶段禁止

- 不修改 `IIoT.CloudPlatform`。
- 不修改 `IIoT.EdgeClient`。
- 不新增 Cloud 端点。
- 不开放 Cloud 写。
- 不开放 Recipe / Recipe version。
- 不开放 `query_cloud_data_readonly`。
- 不新增生产自由 SQL。
- 不保存 token、API Key、连接串、完整 SQL、完整 payload、rows 或敏感上下文。
- 不把 P14.2 表述为 GA 或全面生产上线。

## 完成条件

- emergency stop 重启后仍能恢复并阻断 P12/P13。
- incident、run ledger、readiness assessment 可持久化查询。
- P15 readiness 在缺少 P12 或 P13 completed run 时必须阻断。
- run ledger 和报告只包含 endpoint、source mode、boundary、hash、row count、truncated、approval status、status。
- `scripts/Run-EnterpriseCloudReadonlyProductionOperationsP14_2Acceptance.ps1` 通过并输出：

```text
docs/enterprise-cloud-readonly-production-operations-p14_2-latest.md
```
