# A助理企业级数据源与知识治理 P15 范围冻结

## 阶段定位

P15 是生产只读 Pilot 的规划与授权闸门阶段，不是 P16 真实小范围试用，也不是 GA。

本阶段只把 P14.2 的运营基线整理成可提交授权评审的规划包，包含试点范围、角色权限、数据保留、审批链、回滚、监控、风险和进入 P16 的准入条件。

## 允许范围

- 只修改 `AICopilot`。
- 新增或更新 P15 规划文档、验收脚本、报告和最小只读前端提示。
- 复用 P12/P13/P14.2 已有 evidence，不扩大生产读取能力。
- 输出 P16 前置 blocker 清单，明确哪些工程项必须在真实 Pilot 前关闭。

## 禁止范围

- 不修改 `IIoT.CloudPlatform`。
- 不修改 `IIoT.EdgeClient`。
- 不新增 Cloud endpoint。
- 不开放 Cloud 写。
- 不开放 Recipe / Recipe version。
- 不开放自由 SQL。
- 不开放 `query_cloud_data_readonly`。
- 不把 P15 显示成真实 Pilot 已开始或 GA。
- 不输出 token、API Key、连接串、完整 SQL、raw payload、rows 或敏感上下文。

## P15 默认试点边界

- 试点用户：5-10 人。
- 角色：Admin、TrialManager、Approver、Operator、Viewer。
- Endpoint：`devices`、`capacity_summary`、`device_logs`、`pass_station_records`。
- 数据范围：最近 7 天。
- 默认 `maxRows`：50。
- 产物：Markdown、HTML、PDF、PPTX、XLSX 草稿和 final 审批产物。
- 所有 final 产物必须经过 Final Approval。

## P16 前置 blocker

以下 blocker 未关闭前，不允许进入 P16 真实 Pilot 实施：

1. P12 `ProductionPilotWindow` / `ProductionPilotRun` 持久化。
2. P13 `ProductionControlledPilotIntent` / `ProductionControlledPilotRun` 持久化。
3. final artifact refs 自动回写 `ProductionPilotRunLedger`。
4. P12/P13 rows 保留、脱敏、TTL、下载和 artifact 使用策略明确。
5. P14 operations 权限 smoke 覆盖普通用户拒绝 emergency stop、incident upsert、GA readiness evaluation。
6. 长时间运行和并发验证覆盖多用户、多 endpoint、incident/ledger 并发写入、emergency stop 竞争场景。

## 完成条件

- P15 规划包、范围冻结文档和验收报告存在。
- P14.2 acceptance 继承验证通过。
- scope guard 通过。
- 前端最小入口明确显示“规划与授权闸门 / 非 GA / 非真实 Pilot 执行”。
- P16 blocker 清单完整，并且未关闭时不得输出 `ReadyForP16Execution`。
