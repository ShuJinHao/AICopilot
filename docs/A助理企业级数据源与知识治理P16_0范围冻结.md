# A助理企业级数据源与知识治理 P16.0 范围冻结

## 阶段定位

P16.0 是真实 Pilot 前置工程硬化阶段，不是 GA，不是全面生产上线，也不是扩大生产读取范围。

本阶段只关闭 P15 明确保留的工程 blocker，让 P12/P13 生产只读 Pilot 在进入小范围真实执行评审前具备可恢复、可审计、可脱敏复核的工程基础。

## 修改边界

- 只修改 `AICopilot`。
- 不修改 `IIoT.CloudPlatform`。
- 不修改 `IIoT.EdgeClient`.
- 不新增 Cloud 端点。
- 不开放 Cloud 写。
- 不开放 Recipe / Recipe version。
- 不开放自由 SQL。
- 不开放 `query_cloud_data_readonly`。

## 本阶段允许内容

- 将 P12 `ProductionPilotWindow` / `ProductionPilotRun` 从内存态升级为持久化聚合。
- 将 P13 `ProductionControlledPilotIntent` / `ProductionControlledPilotRun` 从内存态升级为持久化聚合。
- 接入 `AiGatewayDbContext`，补齐 EF configuration、migration、designer、snapshot。
- final artifact 审批锁定后自动回写 `ProductionPilotRunLedger` 的 final artifact refs。
- 明确 production Pilot rows 只允许短期运行态使用，operations ledger 永远 hash-only。
- 增加 rows retention / artifact-use / download policy 的摘要和测试。
- 补齐 operations 权限、emergency stop 并发、ledger/incident 并发硬化测试。

## 数据留存边界

P16.0 的生产 Pilot 运营层只允许保存：

- source mode
- boundary
- endpoint code
- pilot window id
- intent id
- artifact ids
- approval status
- duration
- row count
- truncated
- query hash
- result hash
- status
- error code

P16.0 禁止在 operations ledger、readiness、报告或前端中保存或回显：

- token
- API Key
- 连接串
- 完整 SQL
- raw payload
- rows / raw business records
- 敏感上下文

## 完成条件

- P12/P13 store 可持久化、可重启恢复。
- emergency stop active 时，重启后仍阻断 P12/P13 生产只读 Pilot。
- P12/P13 final artifact refs 能自动回写 operations ledger。
- P15 readiness 不再依赖人工 upsert artifact refs。
- rows retention 策略明确为 hash-only operations ledger。
- 权限和并发硬化测试通过。
- `Run-EnterpriseProductionPilotHardeningP16_0Acceptance.ps1` 生成脱敏验收报告。

## 非完成含义

P16.0 完成不表示已经允许真实 Pilot 自动运行，也不表示 GA。真实 endpoint/token 仍只能在后续明确 Pilot Window、审批链、回滚策略、数据保留策略之后使用。
