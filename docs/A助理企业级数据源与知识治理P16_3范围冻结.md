# A助理企业级数据源与知识治理 P16.3 范围冻结

## 阶段定位

P16.3 是真实 Pilot 执行计划冻结与 5.5 Pro 复核结论落账阶段，不是真实 Pilot 执行，不接真实 endpoint/token，也不是 GA。

本阶段只把 P16.0/P16.2 已形成的工程硬化和执行准入证据整理成可复核的执行计划包。当前 5.5 Pro 复核按 `ReviewPending` 处理，未返回无 Blocker 结论前，不允许进入真实 Pilot 执行。

## 修改边界

- 只修改 `AICopilot`。
- 不修改 `IIoT.CloudPlatform`。
- 不修改 `IIoT.EdgeClient`。
- 不新增 Cloud 端点。
- 不开放 Cloud 写（Cloud write）。
- 不开放 Recipe / Recipe version。
- 不开放自由 SQL。
- 不开放 `query_cloud_data_readonly`。
- 不配置、不调用真实 endpoint/token。

## 本阶段允许内容

- 新增真实 Pilot 执行计划文档。
- 固定 Pilot Window 必填输入：名称、起止时间、负责人、审批人、回滚人、emergency stop 负责人。
- 固定 endpoint allowlist：`devices`、`capacity_summary`、`device_logs`、`pass_station_records`。
- 固定默认边界：最近 7 天、`maxRows=50`、Tool Approval、Final Approval、emergency stop。
- 记录 5.5 Pro 复核状态和 Go/No-Go 状态。
- 新增 P16.3 acceptance 脚本和脱敏报告。

## Go/No-Go 状态

- `BlockedByReview`：5.5 Pro 返回 Blocker，或 GitHub 当前 head CI 未通过。
- `ReadyForLimitedPilotExecutionPlanning`：CI 通过，执行计划包完整，但 5.5 Pro 仍 pending 或仅允许继续规划。
- `ReadyForLimitedPilotExecution`：仅在 5.5 Pro 无 Blocker、Pilot Window 获批、审批链完整、runtime credentials 按批准配置提供后才允许出现。

P16.3 默认只能输出 `ReadyForLimitedPilotExecutionPlanning`，不得自动输出 `ReadyForLimitedPilotExecution`。

## 数据与报告边界

P16.3 文档、报告和只读诊断材料只允许记录：

- PR head / CI 状态。
- Pilot Window 输入清单。
- endpoint allowlist。
- 审批链和回滚要求。
- emergency stop 要求。
- record retention 策略摘要。
- hash-only ledger 证据引用。

禁止记录或回显：

- token。
- API Key。
- 连接串。
- 完整 SQL。
- raw payload。
- 原始业务记录。
- 敏感上下文。

## 完成条件

- 范围冻结文档完成。
- 执行计划包完成。
- P16.3 acceptance 脚本生成脱敏报告。
- 报告明确 5.5 Pro 为 `ReviewPending`。
- 报告不得宣称真实 Pilot 已执行、已允许执行或已经 GA。
- GitHub PR #48 当前 head CI success 后，才可把本阶段材料交给 5.5 Pro 继续复核。
