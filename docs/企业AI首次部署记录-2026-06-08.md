本次部署过程中遇到并修正了两个问题：

1. 服务器直接拉 Docker Hub 基础镜像超时，所以基础镜像改为先推到 Harbor 的 `enterprise-ai` 项目，再由服务器从 Harbor 拉取。

## 首次部署阶段

### 阶段 1：AI 独立上线

- `CloudOidc` 关闭。
- `CloudAiRead` 关闭。
- `CloudReadonly.Mode=Disabled`。
- 启动 `postgres`、`eventbus`、`qdrant`、`migration`、`httpapi`、`dataworker`、`ragworker`、`webui`。
- 对外访问 `http://10.98.90.154:82`。

### 阶段 2：接 Cloud 只读数据

Cloud 端后续需要签发 AI 服务账号 JWT，要求只读权限，不允许 Cloud 写权限。

AICopilot 侧开启：

```env
CLOUD_READONLY_MODE=Real
CLOUD_READONLY_REAL_ENABLED=true
CLOUD_READONLY_REAL_ALLOW_PRODUCTION_READ=true
CLOUD_AI_READ_ENABLED=true
CLOUD_AI_READ_BASE_URL=http://10.98.90.154:81
CLOUD_AI_SERVICE_ACCOUNT_TOKEN=<Cloud 签发的 AI 只读服务账号 JWT>
```

### 阶段 3：接 Cloud OIDC

当前阶段使用隔离内网 HTTP OIDC，不购买公网 HTTPS 证书。Cloud 侧必须打开 `ALLOW_INTRANET_HTTP_OIDC=true`，并注册 AI 回调地址：

```text
http://10.98.90.154:82/api/identity/cloud-oidc/callback
```

AICopilot 侧开启：

```env
CLOUD_OIDC_ENABLED=true
CLOUD_OIDC_ISSUER=http://10.98.90.154:81
ALLOW_INTRANET_HTTP_OIDC=true
CLOUD_OIDC_REQUIRE_HTTPS_METADATA=false
CLOUD_OIDC_CLIENT_ID=aicopilot
```

AICopilot Web 镜像构建时必须注入 Cloud 返回入口，`deploy/enterprise-ai/build-and-push.sh` 默认使用：

```text
VITE_CLOUD_PLATFORM_URL=http://10.98.90.154:81
```

Cloud 切到正式 HTTPS 后，再关闭 `ALLOW_INTRANET_HTTP_OIDC` 并把 issuer/callback 改成 HTTPS。

## 2026-06-08 Cloud OIDC 内网 HTTP 联调修复

本次重新部署 Cloud 和 AICopilot 后，AICopilot 本地账号登录、初始化状态和 Web 入口均正常；Cloud OIDC challenge 初始返回 `500`。根因在 Cloud OIDC discovery/JWKS 反代链路，Cloud 修复后 AICopilot challenge 变为 `302`。

随后发现 HTTP 内网模式还有一个回调风险：OpenIdConnect 默认 nonce/correlation cookie 带 `Secure; SameSite=None`。在 `http://10.98.90.154:82` 场景下，浏览器可能不回传这些 cookie，导致 Cloud 登录回调失败。

代码修复：

1. `CloudOidcOptions.UseHttpCompatibleRemoteCookies()`：
   - 仅当 `ALLOW_INTRANET_HTTP_OIDC=true`
   - 且 issuer 是允许的内网 HTTP 地址时返回 true
2. `HttpApiAuthenticationConfiguration`：
   - HTTPS 生产模式仍保持 `Secure + SameSite=None`
   - 内网 HTTP OIDC 模式改为 `CookieSecurePolicy.SameAsRequest + SameSite=Lax`
3. `CloudOidcLoginTestsControllerSecurity` 补充断言，防止以后回退。
4. `src/vues/AICopilot.Web/nginx.conf.template` 将 API 反代的 `Host`、`X-Forwarded-Host` 改为 `$http_host`，保留外部访问端口 `:82`。
