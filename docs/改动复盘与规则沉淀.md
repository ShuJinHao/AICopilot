## 2026-06-29 线上 Agent artifact workspace 可写路径修复

- 改动范围：AICopilot 后端运行镜像、服务器 compose、`.env.example`、direct Cloud readonly DB 启用 workflow、部署说明、部署模板安全测试和项目规则。
- 改动原因：线上 Plan 确认执行进入 Agent 链路后失败，错误为 `Access to the path '/app/AICopilot' is denied.`；根因是容器以非 root 用户运行，而 `ArtifactWorkspace:RootPath` 未在生产 compose 显式配置，运行时回落到容器内不可写默认路径。
- 影响面：只影响 AICopilot 容器内文件上传根目录和 Agent artifact workspace 根目录；不修改 CloudPlatform、不写 Cloud 业务数据、不改变 DataAnalysis `CloudReadOnly` 只读 SQL guard、白名单表或真实 Cloud DB 连接边界。
- 关键实现：后端 runtime image 预创建并授权 `/var/lib/aicopilot/storage` 与 `/var/lib/aicopilot/artifact-workspaces`；compose 挂载 `enterprise-ai-aicopilot-data` 命名卷到后端服务并注入 `FileStorage__RootPath` / `ArtifactWorkspace__RootPath`；direct enable workflow 在重建后输出两个路径及可写状态；部署说明记录非 root 容器不得依赖 `/app` 或 `LocalApplicationData` 默认路径。
- 规则提炼：形成长期规则，已写入 `AGENTS.md`：容器部署必须显式配置并挂载可写的 `FileStorage:RootPath` 和 `ArtifactWorkspace:RootPath`，不得依赖容器内 `LocalApplicationData` 默认路径或 `/app` 目录写入运行产物。

## 2026-06-29 真实 Cloud DB 只读 Text-to-SQL 接通

- 改动范围：AICopilot DataAnalysis 只读治理、语义映射、语义 SQL 生成、AiGateway runner、Migration worker 数据源注册、服务器 compose/workflow、跨 compose 只读 Docker 网络、本机 secret 脚本、部署说明和相关测试。
- 改动原因：内部 AI 项目开发验证需要直接读取真实 Cloud PostgreSQL 做 Text-to-SQL 效果测试，不走 Simulation，不依赖 Cloud AiRead 插件链路，不写 Cloud 业务数据。
- 影响面：新增 DataAnalysis `CloudReadOnly` 真实只读数据源注册路径；真实 Cloud 查询只允许 `devices`、`device_logs`、`hourly_capacity`、`pass_station_records` 四类白名单表；拒绝敏感字段、通配投影、非白名单表和写 SQL；旧 Simulation 默认链路保持关闭条件和显式标识。
- 关键实现：本机 `scripts/Set-AICopilotCloudReadOnlyDbSecret.sh` 从 Mac 读取只读连接串并写入 GitHub production secret；`scripts/Provision-AICopilotCloudReadOnlyDbRole.sh` 可生成随机密码、写入 GitHub secrets 并触发受控 workflow；`aicopilot-provision-cloud-readonly-db-role` 需要确认词，只创建/轮换专用 readonly role，只授四张白名单表 SELECT，并可随后启用 AICopilot direct DB；`aicopilot-enable-direct-cloud-readonly-db` workflow 写服务器 `.env`、备份并重建 AICopilot 服务；部署脚本创建外部 Docker 网络 `enterprise-ai-cloud-readonly`，直连 DB 模式启用时把 Cloud compose 的 `deploy/postgres` 容器接入并设置别名 `cloud-postgres`；migration worker 只注册 AICopilot 自身的 verified `CloudReadOnly` BusinessDatabase，不修改 CloudPlatform。
- 规则提炼：形成长期规则，已写入 `AGENTS.md` 和 `资料/AICopilot业务规则.md`：内部开发可通过 DataAnalysis `CloudReadOnly` 只读数据源直连真实 Cloud PostgreSQL 做 Text-to-SQL 验证，但必须使用已验证只读账号、白名单表字段和只读 SQL guard，不得写 Cloud 业务数据，也不得用 Simulation 冒充真实结果；创建或轮换 Cloud PostgreSQL 只读账号只能通过显式确认的受控自动化执行，只能创建/更新专用 readonly role，不得授予写权限、schema create 权限、superuser、createdb、createrole 或 replication。
