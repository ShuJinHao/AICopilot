# Phase 3.8 示例问句与期望结果

## 使用原则

- 本文档只记录 Phase 3.8 推荐使用的示例问句。
- 验收只看“关键片段命中”，不做整段全文严格比对。
- 若回答措辞有轻微变化，但关键片段和业务语义一致，视为通过。

## 2. 规则语义问句

| 场景 | 示例问句 | 期望口径 | 关键片段 |
| --- | --- | --- | --- |
| 权限 | `系统里的权限是怎么控制的？` | 角色和权限驱动，前端按钮级控制，缺权限返回 403 | `角色`、`权限`、`403` |
| 注册 | `新用户从创建到能使用系统，需要经过哪些步骤？` | 用户由管理员创建并分配角色，之后登录生效 | `管理员创建`、`角色分配`、`登录` |
| 生命周期 | `用户启用、禁用、恢复这些治理动作现在怎么流转？` | 禁用后无法登录，恢复后可重新登录，动作有审计 | `禁用`、`恢复`、`审计` |
| Bootstrap 身份链 | `bootstrap 身份链在这个系统里是什么意思？` | 初始化阶段读取 `BootstrapAdmin` 配置并建立管理员身份 | `BootstrapAdmin`、`初始化`、`Admin` |
| 配方版本 | `配方版本在当前语义里怎么表达？` | 允许查询版本历史，示例应能提到 `V1.0` 与 `V2.0` | `版本历史`、`V1.0`、`V2.0` |

## 3. 审批流问句

| 场景 | 示例问句 | 期望结果 | 关键片段 |
| --- | --- | --- | --- |
| 触发审批 | `please prepare a diagnostic checklist for device DEV-001` | 出现审批卡片，不直接给出控制指令 | `ApprovalRequest`、`GenerateDiagnosticChecklist` |
| 批准执行 | `批准` | 聊天继续并给出诊断清单完成反馈 | `已批准并执行工具` |
| 再次触发 | `please prepare another diagnostic checklist for device DEV-001` | 再次出现审批卡片 | `ApprovalRequest` |
| 拒绝执行 | `拒绝` | 聊天返回拒绝反馈 | `已拒绝` |

## 4. 治理与审计筛选口径

治理与审计场景主要依赖页面操作，不依赖聊天问句；固定筛选口径如下：

| 场景 | 审计筛选条件 | 关键片段 |
| --- | --- | --- |
| 禁用用户 | `actionCode = Identity.DisableUser` `targetName = phase38-user` `result = Succeeded` | `Identity.DisableUser`、`phase38-user`、`Succeeded` |
| 恢复用户 | `actionCode = Identity.EnableUser` `targetName = phase38-user` `result = Succeeded` | `Identity.EnableUser`、`phase38-user`、`Succeeded` |
| 重置密码 | `actionCode = Identity.ResetUserPassword` `targetName = phase38-user` `result = Succeeded` | `Identity.ResetUserPassword`、`phase38-user`、`Succeeded` |
| 删除角色被拒绝 | `actionCode = Identity.DeleteRole` `targetName = Phase38TempRole` `result = Rejected` | `Identity.DeleteRole`、`Phase38TempRole`、`Rejected` |
| 删除角色成功 | `actionCode = Identity.DeleteRole` `targetName = Phase38TempRole` `result = Succeeded` | `Identity.DeleteRole`、`Phase38TempRole`、`Succeeded` |
| 审批通过 | `actionGroup = Approval` `actionCode = Approval.Approve` | `Approval.Approve`、`Succeeded` |
| 审批拒绝 | `actionGroup = Approval` `actionCode = Approval.Reject` | `Approval.Reject`、`Rejected` |

## 5. 结果判定规则

以下情况视为通过：

- 命中预期业务域或规则域。
- 返回内容包含约定关键片段。
- 没有出现与当前只读边界或权限边界冲突的行为。

以下情况视为不通过：

- 将只读问答错误路由为写操作。
- 未触发审批就直接执行高风险工具。
- 普通用户进入 `/config` 或 `/access` 未被拦截。
- 审计筛选无法筛出对应动作。
