# 联调进入条件与接线步骤

## 进入条件

未来真实联调只能在以下条件全部满足后开始：

- 云端已经能提供统一只读视图或读模型
- [04-云端输入清单.md](/C:/Users/jinha/Desktop/产线系统架构升级/1/AICopilot/资料/Phase40-真实只读源接入准备/04-云端输入清单.md) 已补齐
- [03-真实源差异对照总表.md](/C:/Users/jinha/Desktop/产线系统架构升级/1/AICopilot/资料/Phase40-真实只读源接入准备/03-真实源差异对照总表.md) 已逐项填写
- `ProductionData` 已统一成单一结果集
- `Device.status` 与 `Device.lineName` 已明确可用

只要有一项不满足，就继续停留在准备阶段。

## 固定接线顺序

1. 拿到云端输入
   - 先确认 provider、只读源名称、字段清单、关键字段语义解释都齐了。
2. 填差异表
   - 先在 [03-真实源差异对照总表.md](/C:/Users/jinha/Desktop/产线系统架构升级/1/AICopilot/资料/Phase40-真实只读源接入准备/03-真实源差异对照总表.md) 里把五域逐项标清楚。
   - 只有差异状态能被判为 `Matched` 或 `NeedsNormalize` 的行，才进入下一步处理。
3. 回填真实源模板草稿
   - 以 [appsettings.RealSource.template.json](/C:/Users/jinha/Desktop/产线系统架构升级/1/AICopilot/src/Hosts/AICopilot.HttpApi/appsettings.RealSource.template.json) 为唯一模板。
   - 逐域填写 `DatabaseName`、`SourceName` 或 `FromClause`、`Provider`、`FieldMappings`、允许字段列表。
4. 看语义源状态接口
   - 先检查 `/api/data-analysis/semantic-source/status`
   - 或在配置页语义源状态区检查五域状态
   - 这一阶段先看元数据，不先做聊天联调
5. 五域全 `Ready` 后再做聊天验收
   - 只有五域都满足 `effectiveSourceName` 正确、`providerMatched=true`、`sourceExists=true`、`missingRequiredFields=[]` 时，才进入聊天回归

## 联调时先看什么

先看元数据状态，不先看答案内容：

- `DatabaseNotFound`
- `Disabled`
- `NotReadOnly`
- `ProviderMismatch`
- `SourceNotFound`
- `FieldMismatch`

这几个状态没有清零之前，不进入聊天问答验证。

## 聊天验收最小范围

五域全 `Ready` 后，再按最小范围做聊天验收：

- 设备问答
- 设备日志问答
- 配方问答
- 产能问答
- 生产数据问答
- 规则问答不退化
- 审批链不退化
- 治理链不退化

## 联调红线

- `ProductionData` 未统一前，不进入真实切换
- `Device.status` / `Device.lineName` 缺失时，不进入真实切换
- 不允许为了接真实源而修改聊天主链、语义执行器或增加源侧特判分支

## 失败后的处理顺序

1. 回到状态接口，定位是库、provider、源对象还是字段问题
2. 回到差异表，更新该字段的真实差异状态
3. 如果需要补归一化，在云端侧处理，不在 AICopilot 主链补洞
4. 未恢复到五域全 `Ready` 前，不继续聊天联调
