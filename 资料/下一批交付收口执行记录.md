## 建议 PR 拆分

### PR1：持久化边界与迁移安全

范围：

- IdentityStore migration owner：MigrationWorkApp 执行 `IdentityStoreDbContext` baseline migration。
- Identity destructive migration guard：`MigrateIdentityKeysToGuid` 在 `identity.AspNet*` 已有数据时 fail fast。
- MCP migration 一致性：`InitialMcpServerSchema` 改为 guarded `ALTER TABLE ... SET SCHEMA`，补齐 `McpServerDbContextModelSnapshot`。
- DbContext ownership：AiCopilot 主迁移、Identity runtime、模块 DbContext、Audit/Outbox 边界。
- Architecture / Backend migration tests。

PR 说明必须写清：

- `MigrateIdentityKeysToGuid` 仍是开发期破坏性迁移，会丢弃旧 `public.AspNet*` Identity 数据。
- 已有 `identity.AspNet*` 用户数据会被 SQL guard 拦住，不允许静默删除。
- 本轮不拆 `__EFMigrationsHistory`，这是刻意后置的迁移治理债，不是遗漏。
- Docker-dependent BackendTests 在当前机器被 Docker daemon 阻断，merge 前必须在 Docker 可用环境重跑。

### PR2：AI runtime、前端与旧代码收口

范围：

- AI Gateway runtime、流式聊天、approval resume、final-agent context。
- RAG / DataAnalysis / MCP runtime 整理。
- SQL AST guardrail 与只读执行路径。
- 前端聊天、审批、身份、配置相关页面与 store。
- 删除旧 parser、旧 bootstrap、旧 Services.Common 等历史实现。
- 验收脚本、交付文档、运行手册。

PR 说明重点：

- 说明这批不是新增业务能力，而是把当前 AI runtime 做成可验收、可回滚的交付单元。
- 明确前端与后端协议边界，尤其 SSE error chunk、approval 状态、loading 复位。
- 删除旧代码必须和替代路径一起审查，避免只看 delete diff。
