# AICopilot 业务语义内聚说明

## 目的

本说明用于解释 `AICopilot` 当前 AI 侧业务语义目录如何对应 [业务验收基线说明](./业务验收基线说明.md)。

本阶段不新增业务主题、不改变对外接口，只把已经确认的五域结构化问答与五个 `Policy.*` 主题收口到统一内部语义层。

## 代码入口

- 统一业务语义目录：`src/Services/AICopilot.AiGatewayService/BusinessSemantics/BusinessSemanticsCatalog.cs`
- 路由 prompt 组装：`src/Services/AICopilot.AiGatewayService/BusinessSemantics/IntentRoutingPromptComposer.cs`
- 五域摘要 profile：`src/Services/AICopilot.AiGatewayService/BusinessSemantics/SemanticSummaryProfiles.cs`

## 口径边界

- 结构化只读业务域仍然只有：设备、设备日志、配方、产能、生产数据
- 业务规则主题仍然只有：人员权限、设备注册、设备生命周期、Bootstrap 身份链、配方版本
- 当前回答骨架仍然遵循：先结论，再关键指标或适用条件，再关键记录或禁止放宽边界，最后说明查询范围或规则适用范围

## 对应关系

- `Policy.*` 描述器负责沉淀：业务结论、适用条件、禁止放宽边界、示例问句
- `Analysis.*` 描述器负责沉淀：意图描述、示例问句、Query JSON 示例、路由优先级说明
- `SemanticSummaryProfile` 负责沉淀：五域字段标签、指标标签、亮点记录格式、回答骨架约束

## 维护原则

- 如果业务验收基线新增已确认主题，先更新 [业务验收基线说明](./业务验收基线说明.md)，再补目录注册
- 如果规则尚未确认，不进入业务语义目录
- 不允许为了接入新数据源而在聊天主链或执行器中加语义特判分支
