# AICopilot 评审拆包清单

更新时间：2026-04-24

## 使用方式

本清单用于把当前大变更拆成可审查的能力组。评审时按组阅读，不建议从全量 diff 顺序阅读。每组只判断本组行为是否成立、是否有遗漏测试、是否影响上线回滚。

## 1. 配置治理

- 核心范围：`HttpApi` 配置、`AppHost` 编排、`MigrationWorkApp` 初始化、敏感值脱敏。
- 行为变化：生产默认 `AiGateway:FinalAgentContextStore:Provider=Redis`，开发环境保留 `Memory`；模型、数据库、MCP、RAG 配置从测试种子转向正式配置管理。
- 风险点：生产未提供 `ConnectionStrings__final-agent-context-redis` 会导致 HttpApi 启动失败；开发 JWT Secret 和 AppHost 本地 PG 密码不能进入生产部署模板。
- 回滚点：配置错误优先回滚环境变量或部署模板；不要回滚数据库或删除卷。
- 必看文件：`src/Hosts/AICopilot.HttpApi/appsettings.json`、`src/Hosts/AICopilot.AppHost/AppHost.cs`、`src/Hosts/AICopilot.MigrationWorkApp/Worker.cs`。

## 2. 身份权限

- 核心范围：登录、角色、用户管理、权限校验、审计查询。
- 行为变化：补齐管理接口、审计记录和问题详情返回，支持配置管理所需的身份权限闭环。
- 风险点：错误码与 HTTP 状态必须稳定；审计摘要不能泄露 Secret、连接串、模型密钥。
- 回滚点：若身份管理异常，优先回滚 HttpApi 与 IdentityService；数据库不做破坏性操作。
- 必看文件：`src/Hosts/AICopilot.HttpApi/Controllers/IdentityController.cs`、`src/Services/AICopilot.IdentityService/Commands`、`src/Infrastructure/AICopilot.EntityFrameworkCore/AuditLogs`。

## 3. AI Gateway

- 核心范围：SSE 流、会话历史、Agent 创建、模型模板配置、Token 预算。
- 行为变化：聊天和审批流统一错误兜底；模型或模板缺失稳定返回 `chat_configuration_missing`；消息历史按批持久化，减少孤儿消息。
- 风险点：SSE 协议字段不能变形；异常路径必须结束流并让前端复位 loading。
- 回滚点：若聊天主链路异常，回滚 HttpApi 与 AiGatewayService，不单独回滚前端协议。
- 必看文件：`src/Services/AICopilot.AiGatewayService/Agents/ChatStreamRequest.cs`、`src/Services/AICopilot.AiGatewayService/Agents/ChatAgentFactory.cs`、`src/Vues/AICopilot.Web/src/stores/chatStore.ts`。

## 4. 审批与安全

- 核心范围：人工审批、现场在岗声明、高风险工具闸门、审批审计。
- 行为变化：审批按 session 串行化；重复审批收口为 `approval_already_processed`；现场在岗时间改为 UTC 安全字段；高风险工具执行前要求显式确认。
- 风险点：并发 approval callId 只能执行一次工具；过期在岗声明必须拦截；拒绝审批不能触发工具调用。
- 回滚点：审批逻辑异常时优先回滚 AiGatewayService；Redis 中未完成审批可失效，但不影响已落库历史。
- 必看文件：`src/Services/AICopilot.AiGatewayService/Approvals`、`src/Core/AICopilot.Core.AiGateway/Aggregates/Sessions/Session.cs`、`src/Tests/AICopilot.BackendTests/Phase43SafetyQualityTests.cs`。

## 5. Redis 上下文

- 核心范围：`IFinalAgentContextStore`、Redis 序列化、审批恢复上下文。
- 行为变化：新增 Redis store，AppHost 和生产配置默认走 Redis，支持 Chat 和 Approval 落在不同实例。
- 风险点：序列化必须覆盖 pending approval、tool arguments、agent session；Redis 连接失败应在启动阶段暴露。
- 回滚点：本地可切回 Memory；预发和生产不要切回 Memory，除非明确接受单实例限制。
- 必看文件：`src/Services/AICopilot.AiGatewayService/Agents/FinalAgentContextStore.cs`、`src/Services/AICopilot.AiGatewayService/Agents/FinalAgentContextSerializer.cs`、`src/Tests/AICopilot.BackendTests/AcceptanceClosureVerificationTests.cs`。

## 6. SQL Guardrail

- 核心范围：SQL AST 校验、只读数据库执行、Text-to-SQL 输出大小限制。
- 行为变化：从关键字黑名单升级为 AST 只读校验；大结果返回截断预览，避免撑爆模型上下文。
- 风险点：SQL Server 与 PostgreSQL 方言都要保留只读查询能力；禁止 DML、DDL、系统操作和绕过注释。
- 回滚点：若误杀查询，优先修 AST 白名单，不回退关键字黑名单。
- 必看文件：`src/Infrastructure/AICopilot.Dapper/Security/AstSqlGuardrail.cs`、`src/Infrastructure/AICopilot.Dapper/DapperDatabaseConnector.cs`、`src/Services/AICopilot.DataAnalysisService/Plugins/DataAnalysisPlugin.cs`。

## 7. 前端聊天体验

- 核心范围：聊天流消费、错误提示、审批卡片、会话历史刷新。
- 行为变化：Error chunk 映射到明确提示；流结束时 loading 复位；审批卡片支持现场在岗声明和重复审批结果。
- 风险点：不能出现静默结束、一直 loading、重复审批无反馈；SSE 协议字段不新增。
- 回滚点：若仅 UI 异常，优先回滚 Web 资产，不动后端和数据库。
- 必看文件：`src/Vues/AICopilot.Web/src/stores/chatStore.ts`、`src/Vues/AICopilot.Web/src/components/chat/ChatWindow.vue`、`src/Vues/AICopilot.Web/src/components/chat/ApprovalCard.vue`。

## 8. 测试与脚本

- 核心范围：验收脚本、Docker/Aspire smoke、migration schema 验证、前端 build。
- 行为变化：完整验收脚本覆盖后端 build、前端 build、diff whitespace、focused tests、Docker、migration/Redis、三条 runtime smoke。
- 风险点：Docker 不可用时不能声称端到端验收完成；预发上线前必须在目标环境重新跑脚本。
- 回滚点：测试脚本只影响交付验证，不作为运行时回滚对象。
- 必看文件：`scripts/Run-AcceptanceClosure.ps1`、`src/Tests/AICopilot.BackendTests/AcceptanceClosureVerificationTests.cs`、`src/Tests/AICopilot.BackendTests/Phase25RuntimeSmokeTests.cs`。
